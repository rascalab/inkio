# 개발 가이드

## 패키지 구조

```text
packages/
  core/         # low-level foundation
  advanced/     # notion-like and integration-heavy extensions
  simple/       # classic WYSIWYG entry point
  editor/       # notion-like entry point
  image-editor/ # optional image editing UI
```

## 역할 분리

- `@inkio/core`
  - primitives, menus, icons, markdown, `ImageBlock`, callout, details/toggle, table, keyboard shortcuts
- `@inkio/advanced`
  - slash command, block handle, mention, hashtag, wiki link, bookmark, comment
- `@inkio/simple`
  - core-only classic editor
- `@inkio/editor`
  - advanced를 포함한 notion-like editor

## 공통 chrome contract

배치 구조는 패키지마다 유지합니다.

- `simple`: classic toolbar-first
- `editor`: notion-like
- `image-editor`: fullscreen overlay

대신 시각 시스템은 공통 surface 토큰을 기준으로 맞춥니다.

- `surface-shell`
  - toolbar, dock, command bar, sidebar
- `surface-panel`
  - popover, bubble menu, floating card, picker
- `surface-field`
  - input, select, textarea, search, segmented control

focus ring, border, blur, shadow, field styling은 가능한 한 이 공용 contract를 따릅니다.

## dependency 정책

- `react`, `react-dom`만 peer
- Inkio가 직접 쓰는 Tiptap runtime은 각 패키지 `dependencies`
- `@inkio/core`는 published package지만 일반 사용자에게 직접 시작점으로 안내하지 않습니다
