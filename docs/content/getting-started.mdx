# 시작하기

이 가이드는 Inkio를 프로젝트에 통합하는 최소 단계만 설명합니다.

## 설치

```bash
npm install @inkio/editor @inkio/extension react react-dom
```

## 스타일 선택

```tsx
import '@inkio/editor/minimal.css';
import '@inkio/extension/style.css';
```

## 기본 사용

```tsx
import { useMemo, useState } from 'react';
import { Editor, getDefaultCoreExtensions } from '@inkio/editor';
import { Callout } from '@inkio/extension/callout';
import { HashTag } from '@inkio/extension/hashtag';
import { SlashCommand } from '@inkio/extension/slash-command';
import { WikiLink } from '@inkio/extension/wikilink';
import '@inkio/editor/minimal.css';
import '@inkio/extension/style.css';

const tags = ['inkio', 'editor', 'react', 'nextjs'];

export function MyEditorPage() {
  const [content, setContent] = useState({
    type: 'doc',
    content: [{ type: 'paragraph' }],
  });

  const extensions = useMemo(
    () => [
      ...getDefaultCoreExtensions({
        placeholder: 'Start typing...',
      }),
      Callout,
      HashTag.configure({
        items: ({ query }) =>
          tags
            .filter((tag) => tag.includes(query.toLowerCase()))
            .map((tag) => ({ id: tag, label: `#${tag}` })),
      }),
      SlashCommand,
      WikiLink,
    ],
    [],
  );

  return (
    <Editor
      content={content}
      extensions={extensions}
      showBubbleMenu
      showFloatingMenu
      onUpdate={setContent}
    />
  );
}
```

`getDefaultInkioExtensions()`는 편의용 full preset입니다. 빠르게 전 기능을 붙일 때는 유용하지만, 시작점으로는 `getDefaultCoreExtensions()`에 필요한 extension만 직접 조합하는 방식을 권장합니다.
`HashTag`는 `#`와 heading markdown shortcut 충돌 때문에 opt-in입니다.
`ImageBlock`은 core 기본 구성에 포함되며, 필요하면 `@inkio/editor`에서 직접 import해서 옵션을 덮어쓸 수 있습니다.

## i18n 전략

Inkio 기본 내장 메시지는 영어(`en`)만 제공합니다.

- `locale`은 `unknown` 입력을 받습니다.
- `locale`은 문자열, Accept-Language 문자열, 배열, `Intl.Locale`, locale-like 객체를 해석합니다.
- 다국어 메시지는 소비자가 `messages` prop으로 주입합니다.
- 해석 실패/메시지 누락 시 영어 기본값으로 폴백합니다.

자세한 규칙은 [`i18n-locale`](/i18n-locale) 문서를 참고하세요.

## Controlled / Uncontrolled

```tsx
import { Editor } from '@inkio/editor';

// controlled
<Editor content={content} onUpdate={setContent} />

// uncontrolled
<Editor initialContent={initialContent} onUpdate={setContent} />
```

`content`와 `initialContent`는 동시에 사용할 수 없습니다.

## 트러블슈팅

### 스타일이 안 보일 때

스타일 엔트리를 반드시 import했는지 확인하세요.

### 확장 UI가 깨질 때

`@inkio/extension/style.css`가 import되어 있는지 확인하세요. `SlashCommand`, `HashTag`, `BlockHandle`, `Comment`, `ImageEditorModal`에 필요합니다.

### 이미지 편집 UI가 깨질 때

`@inkio/extension/style.css`가 import되어 있는지 확인하세요.

### SSR 오류(`window is not defined`)

에디터는 클라이언트 컴포넌트에서만 사용해야 합니다. Next.js App Router라면 파일 상단에 `'use client';`를 선언하세요.
